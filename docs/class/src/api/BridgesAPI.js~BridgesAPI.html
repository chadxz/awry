    Add a channel to a bridge.

    Params:

    NameTypeAttributeDescription
    params object
    params.bridgeId string

    The identifer of the bridge to add the channels to.
    params.channel string | Array<string>

    The channel or list of channels to add to the bridge.

    • optional

    Resolves if the channels are successfully added to the bridge. Rejected if one of the channels is not found (status 400), the bridge could not be found (status 404), the bridge is not in the Stasis application (status 409), one of the specified channels is currently recording (status 409), or one of the channels is not in the Stasis application (status 422).

    public play(params: object): Promise<Playback> source

    POST /bridges/{bridgeId}/play

    Start playback of media on a bridge. The media URI may be any of a number of URIs. The currently support URIs are 'sound:', 'recording:', 'number:', 'digits:', 'characters:', and 'tone:'. This operation creates a playback resource that can be used to control the playback of media (pause, rewind, fast-forward, etc.)

    'tone:' playback uri added in Asterisk 12.3

    Params:

    NameTypeAttributeDescription
    params object
    params.bridgeId string

    The identifier of the bridge to play media into.
    params.media string | Array<string>

    The media's URI to play. Allows multiple media to be passed since Asterisk 14.0
    params.playbackId string
    • optional

    The playback identifier to attach to the Playback instance. Param available since Asterisk 12.2
    params.lang string
    • optional

    For sounds, the language of the sound to play.
    params.offsetms number
    • optional
    • default: 0

    The number of milliseconds to skip from the beginning of the media before starting playback. Allowed range: 0+
    params.skipms number
    • optional
    • default: 3000

    The number of milliseconds to skip for forward/reverse operations. Allowed range: 0+

    Return:

    Promise<Playback>

    Resolved with the Playback instance when the playback is successfully started. Rejects when the bridge is not found (status 404) or the bridge is not in the Stasis application (status 409).

    public record(params: object): Promise<LiveRecording> source

    POST /bridges/{bridgeId}/record

    Start a recording. This records the mixed audio from all channels participating in this bridge.

    Params:

    NameTypeAttributeDescription
    params object
    params.bridgeId string

    The identifier of the bridge to record.
    params.name string

    The recordings filename.
    params.format string
    • optional
    • default: wav

    The format to encode the audio in.
    params.maxDurationSeconds number
    • optional
    • default: 0

    The maximum duration of the recording, in seconds. Pass 0 for no limit. Allowed range: 0+
    params.maxSilenceSeconds number
    • optional
    • default: 0

    The maximum duration of silence, in seconds. Pass 0 for no limit. Allowed range: 0+
    params.ifExists string
    • optional
    • default: fail

    The action to take if a recording with the same name already exists. Allowed values: 'fail', 'overwrite', or 'append'.
    params.beep boolean
    • optional
    • default: true

    Play a beep when recording begins.
    params.terminateOn string
    • optional
    • default: none

    DTMF input to terminate recording. Allowed values: 'none', 'any', '*', or '#'.

    Return:

    Promise<LiveRecording>

    Resolves with the created LiveRecording instance if the recording is successfully started. Rejects if invalid parameters are passed (status 400), the bridge is not found (status 404), the bridge is not in the Stasis application (status 409), a recording with the same name already exists on the system and cannot be overwritten because it is already in progress or the ifExists param is set to 'fail' (status 409), or the specified format is unknown (status 422).
